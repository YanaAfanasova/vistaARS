# Проект «АРС — Анестезиологическая рабочая станция» (VISTAMED)

## Общее описание

Проект представляет собой **макет (proof‑of‑concept)** интерфейса для автоматизированного рабочего места анестезиолога-реаниматолога. Разработан для демонстрации заказчику и последующей передачи в промышленную разработку.

**Основные цели макета:**

- Визуализация ключевых этапов ведения пациента в периоперационном периоде.
- Отображение лабораторных данных, плана лекарственной терапии, информированных согласий.
- Обеспечение быстрого доступа к критической информации (газы крови, гемостаз, инфекционный статус).
- Единый стиль и интуитивно понятная навигация для врачей.

---

## Структура проекта

Проект состоит из **статических HTML-страниц**, объединённых общим CSS и JavaScript. Все файлы расположены в корневой папке и не требуют серверной части для просмотра (кроме случаев, когда нужен локальный сервер для корректной загрузки модулей).

```
project/
├── styles.css                  # Единый файл стилей для всех страниц
├── index.html                  # Главная страница-заглушка (перенаправление)
├── anesthesia-card.html        # Карта анестезии (временная шкала, инфузии, показатели)
├── primary-examination.html    # Первичный осмотр анестезиолога (конструктор документа)
├── consents.html               # Формирование информированного согласия
├── analyses.html               # Лабораторный мониторинг (два режима)
├── plan.html                   # План лекарственного сопровождения (вкладка)
├── examination-as-needed.html  # Осмотр по необходимости (заглушка)
├── ns-and-pv.html              # Получение НС и ПВ (заглушка)
├── protocol-anesthesia.html    # Протокол анестезии (заглушка)
├── protocol-ic.html            # Протокол ИК (заглушка)
└── README.md                   # Этот файл
```

## Функциональные блоки

### 1. Шапка и панель пациента (общие для всех страниц)

- **Верхняя зелёная панель** — логотип, ФИО врача, специальность, кнопка возврата к истории болезни.
- **Горизонтальная панель пациента** — отображает ключевые данные: ФИО, возраст, антропометрию, диагноз, аллергии. Панель можно свернуть/развернуть.

---

### 2. Карта анестезии (`anesthesia-card.html`)

**Назначение:** Центральный экран для мониторинга и регистрации событий во время операции.

**Возможности:**
- Временная шкала от 09:00 до 12:00 с масштабированием (5, 15, 30, 60 мин).
- Отображение **фактических** препаратов (разовые маркеры, инфузионные полосы с изменяемой толщиной для Ремифентанила и паузой).
- **Плановые** препараты (полупрозрачные полосы/маркеры) — переносятся из вкладки «План».
- **Регистрация введения** из плана по клику на плановый элемент.
- **Показатели состояния** (пульс, АД, SpO₂, ЦВД, температура) — клик по ячейке открывает модальное окно ввода.
- **ИМН (инструменты)** — клик по ячейке для ввода названия инструмента.
- **Этапы операции** — добавление через кнопку или клик по ячейке строки этапов (список из 15 предопределённых этапов).
- **Линия текущего времени** и таймер операции.
- **Планирование** — отдельная вкладка с таблицей препаратов (разовые/непрерывные) и кнопкой «Применить план в карту».

---

### 3. Первичный осмотр анестезиолога (`primary-examination.html`)

**Назначение:** Конструктор документа для заполнения первичного осмотра перед операцией.

**Возможности:**
- Таблица с полями: дата, назначивший врач, исполнитель, данные иммуно-серологической лаборатории (группа крови, резус, фенотип, KELL), диагноз, жалобы, анамнез, физикальное исследование, мониторинг, заключение, оценка интубации, курение, ЧДД, АД, ЧСС, вид анестезии, вид операции, ASA, рекомендации.
- Кнопки: «Копировать из предыдущего», «Сохранить шаблон», «Загрузить шаблон» (заглушки).
- **Валидация** при подписании: проверка заполненности всех полей, подсветка незаполненных.
- Модальные окна для подтверждения подписания и удаления.

---

### 4. Информированное согласие (`consents.html`)

**Назначение:** Формирование и подписание информированного добровольного согласия на анестезиологическое обеспечение.

**Возможности:**
- Сбор данных пациента и его законного представителя.
- Выбор типа подписи (пациент/представитель).
- Превью печатной формы согласия в реальном времени.
- Фиксация даты и времени подписания.
- Блокировка полей после подписания, возможность редактирования (с предупреждением).
- Кнопки: «Сформировать согласие», «Сохранить», «Подписать», «Печать», «Скачать» (заглушки).

---

### 5. Лабораторный мониторинг (`analyses.html`)

**Назначение:** Отображение лабораторных данных в двух режимах.

**Режимы:**
- **«Анализы во время операции»** — таблица с показателями КЩС, гемостаза, биохимии по времени (колонки 12:00, 12:30, 13:00 и т.д.). Цветовая индикация отклонений. Time Machine — календарь + выбор времени для смены данных (данные статичны, но структура готова к интеграции).
- **«Анализы за период госпитализации»** — дашборд с блоками:
  - Urgent/КЩС (спидометры pH, pCO₂, pO₂, лактат, электролиты с критическим выделением Ca²⁺).
  - Гемостаз и кровь (Hb/Hct, МНО с индикацией риска кровотечения, тромбоциты).
  - Органотоксичность (почечный и печёночный профили с выделением патологии).
  - Инфекционный статус (дерево культур, матрица антибиотикочувствительности).
  - Мониторинг мочи (эритроциты, бактериурия, белок).
- Smart-фильтр «Только патология» — скрывает нормальные показатели, оставляя критические.
- Time Machine — выпадающий список дат госпитализации.

---

### 6. План лекарственного сопровождения (`plan.html`)

**Назначение:** Составление плана препаратов до операции.

**Возможности:**
- Добавление препаратов (разовых и непрерывных) через модальное окно с полями: название, доза, единица, время/диапазон, скорость, способ введения.
- Таблица с возможностью удаления строк.
- Кнопка «Применить план в карту» — перенос плановых препаратов на временную шкалу карты анестезии.

## Технологии

- **HTML5** — семантическая разметка.
- **CSS3** — кастомные свойства (CSS‑variables), Flexbox, Grid, адаптивная вёрстка.
- **JavaScript (ES6)** — вся интерактивность, управление модальными окнами, фильтрация, переключение режимов, валидация форм.
- **Шрифты** — стандартные системные (Segoe UI, Arial, Times New Roman для печатных форм).
- **Иконки** — эмодзи и текстовые символы (для демонстрации).

## Запуск и просмотр

1. **Локальное открытие:**  
   Просто откройте файл `index.html` или любую другую страницу в современном браузере (Chrome, Edge, Firefox).  
   *Примечание:* некоторые браузеры могут блокировать импорт модулей при локальном запуске — в текущей версии мы используем обычные теги `<script>` для совместимости.

2. **Локальный сервер (рекомендуется):**  
   Для корректной работы всех переходов между страницами и для имитации «живого» проекта используйте простой HTTP-сервер:
   ```bash
   npx http-server
   ```
   или
   ```bash
   python -m http.server 8000
   ```
   Затем откройте в браузере `http://localhost:8000`.

3. **Навигация:**  
   Верхнее меню (вкладки) позволяет переключаться между основными разделами.  
   Вкладки «Согласия», «Первичный осмотр», «Карта анестезии», «Анализы» — активны и демонстрируют прототипы.  
   Остальные вкладки («Осмотр по необходимости», «Получение НС и ПВ», «Протокол анестезии», «Протокол ИК») пока являются заглушками.

## Передача заказчику

1. **Упакуйте проект:**  
   Скопируйте все файлы из папки `project/` в один архив и передайте заказчику.

2. **Демонстрация:**  
   Откройте `index.html` в браузере и покажите ключевые сценарии:
   - Заполнение первичного осмотра и подписание.
   - Планирование лекарственной терапии и её применение на карту.
   - Регистрацию введения препаратов (из плана или по клику на сетку).
   - Просмотр лабораторных данных в двух режимах с использованием Time Machine.
   - Формирование и печать информированного согласия.

3. **Для разработчиков:**  
   В проекте используется централизованный CSS (`styles.css`) – все страницы наследуют единый стиль.  
   Специфичные стили для каждой страницы вынесены в блоки `<style>` внутри соответствующих HTML-файлов.  
   JavaScript-логика написана на чистом JS, легко читается и может быть адаптирована под бэкенд (замена статических данных на API-запросы).

## Планируемые доработки (на основе обратной связи)

- Интеграция с реальной лабораторной информационной системой (ЛИС) для подгрузки динамических данных.
- Подключение к электронной истории болезни (выгрузка данных пациента, автоматическое заполнение полей).
- Добавление правой панели с быстрым вводом показателей и схемой катетеров (по требованию заказчика).
- Возможность редактирования ранее введённых значений.
- Сохранение состояния между сессиями (localStorage / сессия).
- Улучшенная печать с поддержкой всех документов в формате A4.
