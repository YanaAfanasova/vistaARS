# Проект «АРС — Анестезиологическая реанимационная система» (ООО "Виста")

## Общее описание

Проект представляет собой **макет (proof‑of‑concept)** интерфейса для автоматизированного рабочего места анестезиолога-реаниматолога. Разработан для демонстрации заказчику и последующей передачи в промышленную разработку.

**Основные цели макета:**

- Визуализация ключевых этапов ведения пациента в периоперационном периоде.
- Отображение лабораторных данных, плана лекарственной терапии, информированных согласий.
- Обеспечение быстрого доступа к критической информации (газы крови, гемостаз, инфекционный статус).
- Единый стиль и интуитивно понятная навигация для врачей.
- Формирование печатной формы по приказу Минздрава РФ № 530н с возможностью просмотра и подписания электронной подписью (демонстрационный режим).

---

## Структура проекта

Проект состоит из **статических HTML-страниц**, объединённых общим CSS и JavaScript. Все файлы расположены в корневой папке и не требуют серверной части для просмотра (кроме случаев, когда нужен локальный сервер для корректной загрузки модулей).

```
project/
├── .git/                         # Система контроля версий
├── .gitattributes                # Настройки Git
├── styles.css                    # Единый файл стилей для всех страниц
├── common.js                     # Общий JavaScript (вспомогательные функции)
├── print-utils.js                # Модуль сбора данных и генерации печатной формы 530н
├── print-form.html               # Самостоятельная страница для просмотра/печати формы 530н
├── README.md                     # Этот файл
│
├── consents.html                 # Информированное согласие (конструктор)
├── anesthesia-primary-exam.html  # Первичный осмотр анестезиолога (конструктор)
├── examination-as-needed.html    # Осмотр по необходимости (конструктор)
├── ns-and-pv.html                # Получение НС и ПВ (протокол назначения НСПВ)
├── anesthesia-card.html          # Карта анестезии (временная шкала, инфузии, показатели, ИВЛ, баланс)
├── protocol-anesthesia.html      # Протокол анестезиологического пособия (конструктор)
├── protocol-ic.html              # Протокол искусственного кровообращения (конструктор)
├── analyses.html                 # Лабораторный мониторинг (два режима)
```

## Функциональные блоки

### 1. Шапка и панель пациента (общие для всех страниц)

- **Верхняя зелёная панель** — логотип, ФИО врача, специальность, кнопка возврата к истории болезни.
- **Горизонтальная панель пациента** — отображает ключевые данные: ФИО, возраст, антропометрию, диагноз, аллергии. Панель можно свернуть/развернуть.

---

### 2. Карта анестезии (`anesthesia-card.html`)

**Назначение:** Центральный экран для мониторинга и регистрации событий во время операции. Полностью соответствует требованиям приказа 530н (адаптированная форма).

**Возможности:**

- **Временная шкала** от 09:00 до 12:00 с масштабированием (5, 15, 30, 60 мин).
- **Отображение фактических препаратов** (разовые маркеры, инфузионные полосы с изменяемой толщиной для Ремифентанила и паузой) с указанием объёма и типа введения (болюс/капельно).
- **Плановые препараты** (полупрозрачные полосы/маркеры) — переносятся из вкладки «План».
- **Регистрация введения** из плана по клику на плановый элемент (открывается модальное окно с полями: доза, объём, скорость, способ введения).
- **Показатели состояния** — пульс, АД, SpO₂, ЦВД, температура, CO₂ (добавлен). Клик по ячейке открывает модальное окно ввода.
- **ИМН (инструменты)** — клик по ячейке для ввода названия инструмента.
- **Этапы операции** — добавление через кнопку или клик по ячейке строки этапов (список из 15 предопределённых этапов).
- **Анестезиологические этапы** — дополнительная строка с буквенными обозначениями (А, Вв, Т, Пв, Э, Д, Б) по приказу 530н. Клик по ячейке открывает окно выбора нескольких меток.
- **Линия текущего времени** и таймер операции.
- **Планирование** — отдельная вкладка с таблицей препаратов (разовые/непрерывные) и кнопкой «Применить план в карту».
- **Блок оборудования** (наркозный аппарат, дыхательный контур) и автоматический расчёт длительности операции/анестезии по этапам.
- **Гидробаланс** — поля кровопотери и диуреза.
- **ИВЛ** — сворачиваемый блок с параметрами: режим, FiO₂, ДО, МОД, частота, пиковое давление, ПДКВ, I:E, а также чекбокс «Нуждается в продленной респираторной поддержке».
- **Катетеры** — интерактивная схема тела для установки катетеров (центральный, периферический, артериальный, бедренный) с указанием размера.
- **Печатная форма** — кнопка «Подписать ЭП» вызывает отдельную страницу с формой 530н, заполненной текущими данными. Форма готова к печати или сохранению в PDF.
- Все модальные окна закрываются по клику на фон или крестик.

### 3. Информированное согласие (`consents.html`)

**Назначение:** Формирование и подписание информированного добровольного согласия на анестезиологическое обеспечение.

**Возможности:**
- Сбор данных пациента и его законного представителя.
- Выбор типа подписи (пациент/представитель).
- Превью печатной формы согласия в реальном времени.
- Фиксация даты и времени подписания.
- Блокировка полей после подписания, возможность редактирования (с предупреждением).
- Кнопки: «Сформировать согласие», «Сохранить», «Подписать», «Печать», «Скачать» (заглушки).

---

### 4. Первичный осмотр анестезиолога (`anesthesia-primary-exam.html`)

**Назначение:** Конструктор документа для заполнения первичного осмотра перед операцией.

**Возможности:**
- Таблица с полями: дата, назначивший врач, исполнитель, данные иммуно-серологической лаборатории (группа крови, резус, фенотип, KELL), диагноз, жалобы, анамнез, физикальное исследование, мониторинг, заключение, оценка интубации, курение, ЧДД, АД, ЧСС, вид анестезии, вид операции, ASA, рекомендации.
- Кнопки: «Копировать из предыдущего», «Сохранить шаблон», «Загрузить шаблон», «Копирование полей осмотров» (заглушки, демонстрируют интерфейс).
- **Валидация** при подписании: проверка заполненности всех полей, подсветка незаполненных.
- Модальные окна для подтверждения подписания и удаления.

---

### 5. Осмотр по необходимости (`examination-as-needed.html`)

**Назначение:** Конструктор для проведения дополнительных (внеплановых) осмотров пациента в периоперационном периоде.

**Возможности:**
- Набор полей, адаптированный под срочную оценку состояния: дата/время осмотра, назначивший врач, исполнитель, жалобы, кожные покровы, диагноз, телосложение, общее состояние, органы дыхания, ЧДД, тоны сердца, АД (справа/слева), ЧСС, трансфузионный и аллергоанамнез, планируемое вмешательство, вид анестезии, вид операции, оценка ASA, особенности.
- Кнопки управления: «Копировать из предыдущего», «Сохранить шаблон», «Загрузить шаблон», «Копирование полей осмотров» (заглушки).
- **Валидация** при подписании: проверка всех полей, подсветка незаполненных.
- Подписание и закрытие документа, удаление осмотра – с подтверждающими модальными окнами.
- Автоматическая подстановка текущей даты/времени при создании нового осмотра.

### 6. Получение НС и ПВ (протокол назначения наркотических средств и психотропных веществ) — `ns-and-pv.html`

**Назначение:** Оформление назначения и получения наркотических средств и психотропных веществ для проведения анестезии.

**Возможности:**
- Таблица с полями: дата выполнения, назначивший врач, исполнитель, номер назначения, обоснование, цель, описание препарата (наименование, дозировка), количество, номер документа, отделение.
- Кнопки: «Копировать из предыдущего», «Сохранить шаблон», «Загрузить шаблон», «Копирование полей» (заглушки).
- **Валидация** при подписании: проверка всех полей, подсветка незаполненных.
- Подписание и закрытие документа, удаление – с модальными окнами подтверждения.
- Автоматическая подстановка текущей даты/времени.

---

### 7. Протокол анестезиологического пособия (`protocol-anesthesia.html`)

**Назначение:** Детальный протокол проведения анестезии (интраоперационный учёт).

**Возможности:**
- Обширный конструктор с более чем 50 полями, охватывающими:
  - Общие сведения (дата, врачи, операционная, хирурги, анестезиологи, медсёстры).
  - Премедикацию, вид анестезии, положение пациента.
  - Временные метки (начало анестезии, начало операции, конец операции).
  - Параметры индукции, интубации, ИВЛ, вентиляции.
  - Поддержание анестезии (гипнотики, ингаляционные анестетики, опиоиды, миоплегия).
  - Гемодинамику, инфузионную терапию, кровопотерю, диурез, баланс жидкостей.
  - Особенности, пробуждение, декураризация.
- Кнопки управления, валидация, подписание и удаление – аналогичны другим конструкторам.
- Данные предзаполнены демонстрационными значениями для наглядности.

---

### 8. Протокол искусственного кровообращения (`protocol-ic.html`)

**Назначение:** Документирование параметров искусственного кровообращения во время операций на открытом сердце.

**Возможности:**
- Специализированные поля: перфузиолог, медсестра-перфузионист, таблица скоростей, аппарат ИК, температура, временные метки (начало ИК, пережатие аорты, конец ИК).
- Параметры оксигенации, канюли, дренажи.
- Объёмы первичного заполнения и коррекции, диурез, баланс за ИК.
- Кардиоплегия (тип, состав, периодичность, объём).
- Восстановление сердечной деятельности, вспомогательное кровообращение, отключение от ИК.
- ВИК (время ИК) и ВПА (время пережатия аорты) – автоматически рассчитываются по временным меткам (в статике предзаполнены).
- Полная валидация, подписание, удаление.

---

### 9. Лабораторный мониторинг (`analyses.html`)

**Назначение:** Отображение лабораторных данных в двух режимах.

**Режимы:**
- **«Анализы во время операции»** — таблица с показателями КЩС, гемостаза, биохимии по времени (колонки 12:00, 12:30, 13:00 и т.д.). Цветовая индикация отклонений. Time Machine — календарь + выбор времени для смены данных (данные статичны, но структура готова к интеграции).
- **«Анализы за период госпитализации»** — дашборд с блоками:
  - Urgent/КЩС (спидометры pH, pCO₂, pO₂, лактат, электролиты с критическим выделением Ca²⁺).
  - Гемостаз и кровь (Hb/Hct, МНО с индикацией риска кровотечения, тромбоциты).
  - Органотоксичность (почечный и печёночный профили с выделением патологии).
  - Инфекционный статус (дерево культур, матрица антибиотикочувствительности).
  - Мониторинг мочи (эритроциты, бактериурия, белок).
- Smart-фильтр «Только патология» — скрывает нормальные показатели, оставляя критические.
- Time Machine — выпадающий список дат госпитализации.

---

### 10. Печатная форма (форма 530н) — `print-form.html` + `print-utils.js`

**Назначение:** Самостоятельная страница для просмотра и печати документа, заполненного данными из карты анестезии, в полном соответствии с приказом Минздрава РФ от 05.08.2022 № 530н.

**Возможности:**
- Автоматически собирает данные из основной карты (пациент, оборудование, ИВЛ, гидробаланс, этапы, препараты, мониторинг, лаборатория, подписи).
- Отображает их в структурированном виде, приближенном к официальному бланку (с таблицами для мониторинга, лаборатории, этапов и т.д.).
- Содержит кнопки «Печать / Сохранить PDF» и «Закрыть».
- Вызывается из основной страницы по кнопке «Подписать ЭП» (данные передаются через `localStorage`).
- Может быть легко адаптирована для интеграции с системами электронной подписи.

## Технологии

- **HTML5** — семантическая разметка.
- **CSS3** — кастомные свойства (CSS‑variables), Flexbox, Grid, адаптивная вёрстка.
- **JavaScript (ES6)** — вся интерактивность, управление модальными окнами, фильтрация, переключение режимов, валидация форм, сбор данных для печатной формы.
- **Шрифты** — стандартные системные (Segoe UI, Arial, Times New Roman для печатных форм).
- **Иконки** — эмодзи и текстовые символы (для демонстрации).

## Запуск и просмотр

1. **Локальное открытие:**  
   Просто откройте любой HTML-файл в современном браузере (Chrome, Edge, Firefox).  
   *Примечание:* для корректной работы перехода на печатную форму (`print-form.html`) и использования `localStorage` все файлы должны быть в одной папке.

2. **Локальный сервер (рекомендуется):**  
   Для корректной работы всех переходов между страницами и для имитации «живого» проекта используйте простой HTTP-сервер:
   ```bash
   npx http-server
   ```
   или
   ```bash
   python -m http.server 8000
   ```
   Затем откройте в браузере `http://localhost:8000`.

3. **Навигация:**  
   Верхнее меню (вкладки) позволяет переключаться между основными разделами.  
   Все вкладки содержат полноценный функционал (конструкторы документов, дашборды, карта анестезии).  
   Таким образом, макет полностью демонстрирует все этапы работы анестезиолога: от осмотра и согласия до интраоперационного протокола, лабораторного мониторинга и формирования финальной печатной формы 530н.

## Дополнительные возможности

- **Адаптивность** — интерфейс корректно отображается на экранах разного размера (от планшетов до широких мониторов).
- **Демонстрационные данные** — большинство форм заполнены тестовыми значениями для наглядности.
- **Готовность к интеграции** — все кнопки сохранения, печати, подписания имеют заглушки, которые легко заменить на реальные API-вызовы.
